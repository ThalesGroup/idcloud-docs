D1Task

Objective-C

@interface D1Task : NSObject

Swift

class D1Task : NSObject

Main class for D1 SDK use cases.

Note

It is recommended to initialize D1Task as soon as the application is launched or the earliest possible time.

Note

Risk API contains session ID internally. The session ID is used by backend to identify the user’s activity. The lifecycle of session ID is linked with D1Task lifecycle, i.e. one D1Task instance use same session ID for all the use cases.

  • -startAnalyzeWithParams:completion:

    Start monitoring of events for the current page. This method should be called at the start of the page or when the app resumes into foreground. e.g. viewWillAppear, UIApplicationDidBecomeActiveNotification.

    Category - Risk API

    Note

    Other Risk API -[D1Task pauseAnalyze], -[D1Task stopAnalyzeWithCompletion:] should only be called after completionBlock succeeds.

    Note

    Possible error: D1ErrorCode.D1ErrorRisk, caused by no internet connection, incorrect risk credentials.

  • -stopAnalyzeWithCompletion:

    Stop monitoring the events for the current page. This method should be called to get the risk payload.

    Category - Risk API

    Note

    Integrator should send the risk payload to their backend, to be used in risk assessment call to D1 backend.

    Note

    Possible error: D1ErrorCode.D1ErrorRisk, caused by incorrect risk credentials.

    Declaration

    Objective-C

    - (void)stopAnalyzeWithCompletion:
    (nonnull void (^)(NSMutableData *_Nullable,
                      NSError *_Nullable))completionBlock;

    Swift

    func stopAnalyze(completion completionBlock: @escaping (NSMutableData?, Error?) -> Void)

    Parameters

    completionBlock

    block to be called when it finishes. It returns on the main thread. When it succeeds, it return risk payload. Otherwise, it returns error details. This completionBlock returns risk payload in NSMutableData. The data is already in JSON format. Integrator can use JSON conversion method like [NSJSONSerialization JSONObjectWithData:options:error] to convert to NSDictionary.
