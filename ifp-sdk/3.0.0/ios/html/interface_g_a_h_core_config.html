Detailed Description

This class will provide mechanism to set configurable setting in RiskEngine core module. This config object has to be passed as first param to initialization() For each configurable setting there should default value, which need to consider if value is not set from App.

◆ setSignalCollectionGlobalTimeout:

- (void) setSignalCollectionGlobalTimeout: (NSUInteger)  timeout

@dsicussion This method is to set global configuration timeout in milliseconds. RE SDK should be configured with the maximum global timeout for collecting all signals listed in the policy so that total signal collection will not take more time than this maximum global configuration timeout. Signals that are not collected within global time will be sent to RE backend as "status timeout" (524 timeout error code). Default timeout value is 4000 milliseconds and it is recommended to set it to minimum 4000 milliseconds to ensure every signal is collected.

Parameters
timeoutto set Global signal collection timeout in milli-seconds.
