Swift

protocol IDCSecurePinPadDelegate

A delegate that contains methods for managing PIN pad UI.

@since 1.0.0

  • -frameForSecurePinPad:

    Called prior to drawing the secure PIN pad.

    This delegate serves as an input to generate specific frame of the secure PIN pad.

    Frame initializations are final and MUST NOT be modified on runtime.

    @since 1.0.0

    Declaration

    Objective-C

    Parameters

    securePinPad

    The secure PIN pad informing the delegate about the impending authentication.
  • -securePinPad:didReceiveKey:

    Called when a key is pressed on the secure PIN pad.

    Developer should update the UI to reflect the changes.

    Examples include:

    • IDCSecurePinPadKey.IDCSecurePinPadKeyInput - Add a new secure character (⬤) to the text field entry.
    • IDCSecurePinPadKey.IDCSecurePinPadKeyDelete - Remove a secure character from the text field entry.

    • IDCSecurePinPadKey.IDCSecurePinPadKeyDone - PIN entry is validated.

    @since 1.0.0

    The secure PIN pad informing the delegate about the impending authentication.
    key

    Type of key being pressed.
  • -securePinPad:didChangeUsage:

    Called when usage for secure PIN pad changed.

    For example, during enrollment, user need to enter and confirm its PIN. At first, developer will receive IDCSecurePinPadUsage.IDCSecurePinPadUsageEnrollNew. During confirmation UI, developer will receive IDCSecurePinPadUsage.IDCSecurePinPadUsageEnrollConfirm.

  • -securePinPad:didFailVerifyWithRetryCountRemaining:

    Called when authentication failed.

    Retry count limit is 5 before an error with domain IDCErrorDomain and code IDCError.IDCErrorUserLockout is returned.

    Retry count will reset upon successful PIN authentication.

    @since 1.0.0

    Declaration

    Objective-C

    didFailVerifyWithRetryCountRemaining:(NSUInteger)retryCountRemaining;

    Swift

    func securePinPad(_ securePinPad: IDCSecurePinPad!, didFailVerifyWithRetryCountRemaining retryCountRemaining: Any!)

    Parameters

    securePinPad

    The secure PIN pad informing the delegate about the impending authentication.
    retryCountRemaining

    Retry count remaining. On first authentication failed, the value of this property will be 4. On last authentication before IDCError.IDCErrorUserLockout will trigger, the value of this property will be 1.
  • -securePinPad:didFailPinRuleWithError:

    Called during enrollment or change PIN.

    This callback is invoked when the inputted PIN code violates the required PIN rules.
