SecurePinPadDelegate

@objc
public protocol SecurePinPadDelegate

A delegate that contains methods for managing PIN pad UI.

Since

1.0.0
  • frame(for:)

    Called prior to drawing the secure PIN pad. This delegate serves as an input to generate specific frame of the secure PIN pad. Frame initializations are final and MUST NOT be modified on runtime.

    Declaration

    Swift

    func frame(for securePinPad: SecurePinPad) -> CGRect

    Parameters

    securePinPad

    The secure PIN pad informing the delegate about the impending authentication.
  • securePinPad(_:didReceiveKey:)

    Called when a key is pressed on the secure PIN pad. Developer should update the UI to reflect the changes. Examples include: SecurePinPadKey.input - Add a new secure character (⬤) to the text field entry. SecurePinPadKey.delete - Remove a secure character from the text field entry. SecurePinPadKey.done - PIN entry is validated.

    Declaration

    Swift

    func securePinPad(_ securePinPad: SecurePinPad, didReceiveKey key: SecurePinPadKey)

    Parameters

    securePinPad

    The secure PIN pad informing the delegate about the impending authentication.
    key

    Type of key being pressed.
  • securePinPad(_:didChangeUsage:)

    Called when usage for secure PIN pad changed. For example, during enrollment, user need to enter and confirm its PIN. At first, developer will receive SecurePinPadUsage.enrollNew. During confirmation UI, developer will receive SecurePinPadUsage.enrollConfirm.

    Declaration

    Swift

    func securePinPad(_ securePinPad: SecurePinPad, didChangeUsage usage: SecurePinPadUsage)

    Parameters

    securePinPad

    The secure PIN pad informing the delegate about the impending authentication.
    usage

    The current usage.
  • securePinPad(_:didFailVerifyWithRetryCountRemaining:)

    Called when authentication failed. Retry count limit is 5 before an error with domain IDCErrorDomain and code IDCError.userLockout is returned. Retry count will reset upon successful PIN authentication.

    Declaration

    Swift

    func securePinPad(_ securePinPad: SecurePinPad, didFailVerifyWithRetryCountRemaining retryCountRemaining: UInt)

    Parameters

    securePinPad

    The secure PIN pad informing the delegate about the impending authentication.
    retryCountRemaining

    Retry count remaining. On first authentication failed, the value of this property will be 4. On last authentication before IDCError.userLockout will trigger, the value of this property will be 1.
  • securePinPad(_:didFailPinRuleWithError:)

    Called during enrollment or change PIN. This callback is invoked when the inputted PIN code violates the required PIN rules.

    Since

    2.2.0

    The secure PIN pad informing the delegate about the impending authentication.
    error
